# Theorizer Design UX Questions

# **Start every bullet point by asking “How might we…”**

## Setup

* Aid the user in understanding the purpose of Theorizer?  
  * Help the user connect Theorizer to their goals?  
* Signal what kind of questions this system handles well vs poorly, so the user doesn’t waste their time?  
* Walk the user through posing a fruitful question to generate theories?

## Theory creation & progress reporting

* Make long-running processes transparent and steerable rather than feeling like waiting?  
* Aid the user in knowing when the right time is to step in?  
* Give the user insight into the paper collection methods we’ve chosen?  
  * Aid the user in steering this collection?  
* Aid the user in viewing the collection of papers that we’ve selected?  
  * Allow the user to supplement the papers we’ve chosen?  
    * Aid the user in understanding what has been collected, so they can assess what is missing?  
* Explain the purpose of the extraction schema?  
  * Aid the user in steering the schema?  
* Help the user quickly assess the evidence extraction?  
  * Aid the user in adding to the extraction?

## Assessing theories

* Help the user assess the trustworthiness of each theory?  
  * Instill the user with productive skepticism?  
* Aid users in quickly assessing which theory across the set to dive into?  
* Give the user ways to signal valuable theories back to the system?  
  * Use what the user has learned?

## Understanding novelty

* Aid the user in understanding the scope of the novelty assessment?  
* Quickly explain the different aspects of novelty?  
* Allow the user to push back on novelty claims?

## Next steps

* Guide the user to the best next steps?

—

## General thoughts

* Support graceful recovery without forcing the user to restart from scratch  
* Help the user quickly assess and steer every part of the process?  
  * Make steering decisions reversible/comparable  
* Ensure that users feel safe experimenting?  
* Ensure that the user’s time is respected?
